# llmdoc

项目知识库，文档驱动开发（DDD）：先写设计文档，再照着实现。每篇文档保持小巧可用，只记录代码本身表达不了的决策与约定，不追求大而全。

## 约定

- 一个 feature / 一个主题 = 一个文件。
- 实现与文档冲突时，以文档记录的**决策**为准（实现细节以代码为准）；决策变了先改文档。
- 文档过时即删或合并，不留僵尸文档。
- 用最简单直白的话写，别发明术语。写完自查：没读过代码的人能不能看懂每一句？（反例："代际检查"、"拆除置空"——说清楚"检查事件来源的 socket 还是不是当前这个"就好。）
